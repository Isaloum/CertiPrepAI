Always challenge the user's request before implementing. Question whether the proposed fix is the right one, offer alternatives, and explain tradeoffs. Do not just blindly implement what is asked.

**Why:** User explicitly asked for this — "every time I ask you anything you need to challenge me."

**How to apply:** On every task, present at least one alternative approach and a brief rationale for which is better before writing any code. Let the user confirm direction first.
